Changelog

20/7/18
Added Statistics
  • Added 3 pie charts to the statistics page
Modified Database Tables
  • Tables now show the most recent ones at the top of the table and are no longer sorted based on type (was too confusing to find new entries)

New Data Viewers Page
  • Allows you to invite others to view from the database with set queries
  • The page itself explains what it is for and how to use it
  • Still a Work in Progress. Backend has yet do be done and some front end still needs work.
New Changelog page (this page)
  • Just a more tidy way of keeping track of changes to the panel.
Misc Changes
  • Registrations now filter values such as nil, N/A, none, -, nothing, etc, from medical and dietary fields. This means we don't have to go in and edit it ourselves so that the registration doesn't come up in medical/dietary queries.
Bug Fixes
  • Fixed postcodes not being added to addresses in the database. This bug was only introduced recently and only affects a few registrations
  • Fixed being able to register without a last name
18/7/18
New Dashboard
  • Shows all recent registrations within the last week
  • Shows a pie chart of the total registrations
  • Shows a line graph of the number of registrations we are getting per day
  • All things shown are live and are updated when you refresh the page
Database Table Improvements
  • Database table now checks every 10 minutes if there has been an update to the table, and if there has been, it updates itself
15/7/18
Database Table Improvements
  • Table now shortens long pieces of data so the size of the table doesn't overflow. Can be clicked on to show the full entry, and can be clicked again to hide it once more.
  • Ability to increase field widths of particular fields.
  • Increased default height of table
14/7/18
Bug Fixes
  • Fixed registrations not sending out emails and not taking users to the thank you page.
